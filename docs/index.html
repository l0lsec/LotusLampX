CLI

Terminal
1
Clone the repo
git clone https://github.com/l0lsec/LotusLampX.git
2
Set up a venv
cd LotusLampX/cli && python -m venv .venv && source .venv/bin/activate
3
Install deps
pip install -r requirements.txt
4
Scan for devices
python main.py scan --name MELK
5
Control
python main.py --all on

Requirements: Python 3.10+, Bluetooth adapter. macOS (CoreBluetooth) or Linux (BlueZ 5.43+).

CLI cheatsheet

# Scan for devices
python main.py scan --name MELK

# All devices at once
python main.py --all on
python main.py --all off
python main.py --all color 255 0 128
python main.py --all brightness 200

# Single device
python main.py -d DEVICE_ADDR on
python main.py -d DEVICE_ADDR color 0 255 0

# Effects
python main.py --all mode 1            # jump
python main.py --all mode 3            # breathe
python main.py --all speed 150         # medium speed

# Scenes
python main.py --all scene 10          # disco
python main.py --all scene 16          # ocean
python main.py --all scene 154         # meteor

# White temperature
python main.py --all warm 255 0        # full warm
python main.py --all warm 0 255        # full cold

# Mic reactive
python main.py --all mic on
python main.py --all sensitivity 200

# Timer & sensor
python main.py --all timer 300         # off in 5 min
python main.py --all sensor on --delay 60

mic

python main.py --all mic on|off

sensitivity

python main.py --all sensitivity 0-255

scene

python main.py --all scene ID

timer

python main.py --all timer SECONDS

sensor

python main.py --all sensor on|off --delay N

reset

python main.py -d ADDR reset

debug

python main.py -d ADDR debug

Tip: Set LOTUSLAMP_DEVICE as an env var to skip -d every time.

Protocol overview

Reverse-engineered from the Android app's E1Achieve, BlePackagingDevice, and AppConstant classes.

E1 Frame Format

Every command is a fixed 9-byte frame written via GATT with WRITE_TYPE_NO_RESPONSE:

0x7E
LEN
CMD
D0
D1
D2
D3
0x00
0xEF

Header · Command ID · Data bytes · Footer

GATT UUIDs

Service:

0000FFF0-0000-1000-8000-00805F9B34FB

Write characteristic:

0000FFF3-0000-1000-8000-00805F9B34FB

Read / notify:

0000FFF4-0000-1000-8000-00805F9B34FB

Device identification

MELK-OA10 devices are classified as SYMPHONY (type 30) by the app's LPHelper.deviceTypeWithName.

They use plain, unencrypted E1 frames — encryption is only for devices matching the ELK-* identifier pattern.

Platform notes

macOS

Bluetooth must be enabled in System Settings. The terminal needs Bluetooth access permission.

On macOS, bleak uses CoreBluetooth UUIDs as device addresses (e.g. 12345678-ABCD-...) rather than MAC addresses.

Linux

Requires BlueZ 5.43+. On modern kernels (5.10+) no root is needed.

On older setups, grant raw network capability:

sudo setcap cap_net_raw+eip $(readlink -f $(which python3))
